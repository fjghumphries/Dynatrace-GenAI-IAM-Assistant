================================================================================
                         DYNATRACE IAM GROUPS
================================================================================

This document describes all IAM groups created by this configuration.
Groups contain users and are assigned policies to define their access.

Total groups (sample config): 8 (4 BU + 4 Application)
Total groups at scale (10 BUs, 2000 Applications): 4,020

================================================================================
GROUP HIERARCHY OVERVIEW
================================================================================

Account Level
│
├── BU-Level Groups (Access to ALL data within a Business Unit)
│   ├── bu1-Admins ─────► Full admin access bu1 (Std User + Admin Features + scoped data)
│   ├── bu1-Users ──────► Read access to all bu1 data (Standard User + scoped data)
│   ├── bu2-Admins ─────► Full admin access bu2 (Std User + Admin Features + scoped data)
│   └── bu2-Users ──────► Read access to all bu2 data
│
└── Application-Level Groups (Access only to specific Application)
    ├── petclinic01-Admins ──► Read data + write settings for PetClinic 01 (bu1)
    ├── petclinic01-Users ───► Read-only access to PetClinic 01 (bu1)
    ├── petclinic02-Admins ──► Read data + write settings for PetClinic 02 (bu2)
    └── petclinic02-Users ───► Read-only access to PetClinic 02 (bu2)


================================================================================
SECTION 1: BUSINESS UNIT LEVEL GROUPS
================================================================================

These groups provide access to ALL data within a specific Business Unit,
across all applications and stages (prod, dev, test).

--------------------------------------------------------------------------------
1.1 bu1-Admins
--------------------------------------------------------------------------------
Name:           bu1-Admins
Base Policy:    Standard User + Admin Features (custom)
Description:    Administrators for Business Unit 1.
                Full access to all bu1 observability data and settings.
                Settings write is scoped via boundary — admins can only
                modify settings on entities with bu1 security context.

Data Scope:     All data where dt.security_context starts with "bu1-"
                Captures: bu1-prod-*, bu1-dev-*, bu1-test-*
                Future-proof: new applications automatically included

Capabilities:
    ✓ View all bu1 monitoring data (logs, metrics, traces, events, bizevents)
    ✓ View entities scoped to bu1
    ✓ Read/write settings for bu1 entities ONLY (scoped by boundary)
    ✓ Full automation admin (create, edit, run, administer workflows)
    ✓ Create and manage SLOs
    ✓ Extensions read/write
    ✓ OpenPipeline write
    ✓ Documents (full CRUD), Davis AI, Segments
    ✓ App Engine admin
    ✗ Cannot modify settings for entities outside bu1 scope

Typical Users:  bu1 DevOps leads, Platform engineers, SRE managers

--------------------------------------------------------------------------------
1.2 bu1-Users
--------------------------------------------------------------------------------
Name:           bu1-Users
Base Policy:    Standard User (Dynatrace default)
Description:    Users for Business Unit 1. Read access to all bu1 data.

Data Scope:     All data where dt.security_context starts with "bu1-"

Capabilities:
    ✓ View all bu1 monitoring data (logs, metrics, traces, events, bizevents)
    ✓ View entities scoped to bu1
    ✓ Read settings (global - all settings, unscoped)
    ✓ SLO read, Automation read/limited write, Davis AI
    ✓ Documents (full CRUD), Segments read/write
    ✗ Cannot write settings
    ✗ Cannot create automation (beyond SIMPLE type)
    ✗ Cannot manage SLOs

Typical Users:  bu1 Developers, Support engineers, Analysts

--------------------------------------------------------------------------------
1.3 bu2-Admins / bu2-Users
--------------------------------------------------------------------------------
Same structure as bu1 equivalents, scoped to "bu2-" security context prefix.
bu2-Admins also uses Standard User + Admin Features (not Admin User).


================================================================================
SECTION 2: APPLICATION LEVEL GROUPS
================================================================================

These groups provide access ONLY to a specific application within a BU.
More restrictive than BU-level groups.

--------------------------------------------------------------------------------
2.1 petclinic01-Admins
--------------------------------------------------------------------------------
Name:           petclinic01-Admins
Base Policy:    Standard User + SLO Manager (custom)
Description:    Administrators for PetClinic 01 (bu1). Settings write scoped
                to this application.

Data Scope:     PetClinic 01 data across active stages:
                - bu1-prod-petclinic01-*
                - bu1-dev-petclinic01-*

Capabilities:
    ✓ View petclinic01 monitoring data across all stages
    ✓ View entities scoped to petclinic01
    ✓ Read/write settings for petclinic01 entities (scoped by boundary)
    ✓ Create and manage SLOs
    ✓ Automation read + limited write (SIMPLE type via Standard User)
    ✓ Documents (full CRUD), Davis AI, Segments
    ✗ Cannot access other applications
    ✗ Cannot create full automation workflows (BU Admin only)

Typical Users:  PetClinic 01 team leads, Application owners

--------------------------------------------------------------------------------
2.2 petclinic01-Users
--------------------------------------------------------------------------------
Name:           petclinic01-Users
Base Policy:    Standard User
Description:    Users for PetClinic 01 (bu1). Read-only data access.

Data Scope:     Same as petclinic01-Admins

Capabilities:
    ✓ View petclinic01 monitoring data across all stages
    ✓ View entities scoped to petclinic01
    ✓ Read settings (global - all settings, unscoped)
    ✓ SLO read, Automation read, Davis AI, Documents, Segments
    ✗ Cannot write settings
    ✗ Cannot manage SLOs or automation
    ✗ Cannot access other applications

Typical Users:  PetClinic 01 developers, Testers, Support staff

--------------------------------------------------------------------------------
2.3 petclinic02-Admins
--------------------------------------------------------------------------------
Name:           petclinic02-Admins
Base Policy:    Standard User + SLO Manager (custom)
Description:    Administrators for PetClinic 02 (bu2). Settings write scoped
                to this application.

Data Scope:     PetClinic 02 data across active stages:
                - bu2-prod-petclinic02-*
                - bu2-dev-petclinic02-*

Capabilities:   Same as petclinic01-Admins, scoped to bu2-*-petclinic02-*

Typical Users:  PetClinic 02 team leads, Application owners

--------------------------------------------------------------------------------
2.4 petclinic02-Users
--------------------------------------------------------------------------------
Name:           petclinic02-Users
Base Policy:    Standard User
Description:    Users for PetClinic 02 (bu2). Read-only data access.

Data Scope:     Same as petclinic02-Admins

Capabilities:   Same as petclinic01-Users, scoped to bu2-*-petclinic02-*

Typical Users:  PetClinic 02 developers, Testers, Support staff

SECTION 3: GROUP SUMMARY TABLE
================================================================================

+----------------------+--------------+--------------------+----------------+--------+
| Group                | Base Policy  | Data Scope         | Settings Write | SLOs   |
+----------------------+--------------+--------------------+----------------+--------+
| bu1-Admins           | Std+AdminFt  | Full bu1           | Scoped (BU)    | Write  |
| bu1-Users            | Std User     | Full bu1           | None (read ok) | Read   |
| bu2-Admins           | Std+AdminFt  | Full bu2           | Scoped (BU)    | Write  |
| bu2-Users            | Std User     | Full bu2           | None (read ok) | Read   |
+----------------------+--------------+--------------------+----------------+--------+
| petclinic01-Admins   | Std User     | bu1-*-petclinic01  | Scoped (PC01)  | Write  |
| petclinic01-Users    | Std User     | bu1-*-petclinic01  | None (read ok) | Read   |
| petclinic02-Admins   | Std User     | bu2-*-petclinic02  | Scoped (PC02)  | Write  |
| petclinic02-Users    | Std User     | bu2-*-petclinic02  | None (read ok) | Read   |
+----------------------+--------------+--------------------+----------------+--------+

Note: Std+AdminFt = Standard User + Admin Features (custom policy).
      Settings READ is global for all groups (Standard User grants unconditional
      settings:objects:read). Only WRITE is scoped via boundaries.
